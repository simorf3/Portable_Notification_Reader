Portable Notification Reader

Reads your Windows 11 notifications aloud using natural, online neural voices (with an offline fallback), controlled entirely from a small system-tray icon.

Single portable .exe No installer No admin rights Rust Windows 10/11 (64-bit)

It is a ground-up Rust rewrite of the original C# app, redesigned to be 100% portable:

  • A single .exe — no installer, no MSIX package, no admin rights.
  • All settings live in config.json next to the executable. Nothing is written to the registry or AppData. Copy the folder to a USB stick and it runs anywhere.
  • Notifications are read by polling the Windows notification database directly (%LOCALAPPDATA%\Microsoft\Windows\Notifications\wpndatabase.db), so the app needs no package identity and no UserNotificationListener registration — exactly what makes a portable, unpackaged build possible.

Features

  • Speaks new toast notifications as they arrive (polls once per second by default; configurable).
  • Full online neural voice catalogue (Microsoft Edge voices) fetched live, including the South-African voices: en-ZA (Leah, Luke), af-ZA (Adri, Willem) and zu-ZA (Thando, Themba).
  • Voices filtered by your Windows display language by default, with a “Show all languages” toggle to reveal the entire catalogue.
  • Offline SAPI voices are always available and are used automatically when there is no internet.
  • Volume with loudness boost — 0–200%. Values above 100% amplify the audio louder than the system default (online voices only).
  • Speaking speed control with friendly presets (Very slow → Very fast).
  • Pause during calls/meetings — on by default. Keeps the app quiet while any app is using the microphone or camera (Teams, Zoom, Slack Huddles, Meet, Discord…), so it never talks over you. Uses the Windows privacy consent store — no admin rights, no audio/video device is opened.
  • Lower other apps while speaking (audio ducking) — on by default. Briefly turns the volume of other apps (a YouTube video, music, a game) down while a notification is read, then restores it. So a notification is never buried under, or shouted over, your media.
  • Per-app mute — any app that has sent a notification appears in the Mute apps menu and can be muted/unmuted with a click. A sensible default mute list ships out of the box, so common background noise is silent immediately; communication apps (Microsoft Teams, Outlook, WhatsApp…) are not muted.
    • Muted by default: Windows Settings, Microsoft Store, Windows Update, Support Assist; Windows Security / Defender / Avast / Bitdefender; Snipping Tool, Snip & Sketch, Ditto, Lightshot, ShareX; OneDrive, Google Drive, Dropbox; NordVPN, FortiClient; Bluetooth / Swift Pair pairing, NVIDIA GeForce Experience, AMD Software.
    • Matching is by the app’s display name (case-insensitive). These defaults only apply to a fresh config.json; an existing config keeps your own choices.
    • Mute by pattern (regex): to silence a whole family of apps at once — even ones you haven’t seen yet — add a Filter messages rule. Filter rules match the app name as well as the text, so a block rule like (?i)vpn|antivirus|onedrive mutes every matching app.
  • Filter rules — block or allow notifications by substring or regex (top-level Filter messages…).
  • Text filtering and replacement — remove or replace words/phrases before speaking (plain or regex). Leave the replacement blank to simply delete the matched text.
  • Emoji handling — toggle Speak emojis to hear emoji meanings (🎉 → “party popper”), or leave it off to strip them silently. Full Unicode CLDR coverage including multi-part sequences.
  • Shorthand expansion — common chat abbreviations (lol, brb, omg…) are expanded before speaking.
  • Diagnostics log — every notification and what the app decided to do with it (spoken, muted, filtered, skipped…) is written to notification-reader.log. Open it any time from Open diagnostics log in the tray menu.
  • Smart speech shaping: the app name is never read aloud; WhatsApp group names are skipped (only “Sender: message” is read); URLs are replaced with friendly “domain link” text.
  • Start automatically at sign-in — on first run the app offers to add itself to your Windows Startup folder (asked only once; skipped if a shortcut already exists).
  • Built-in help — this page is compiled into the exe; open it any time from About / Help in the tray menu (single portable file, nothing extra to keep).
  • Sensible defaults — noisy system sources (Windows Security, Snipping Tool, Windows settings…) are muted out of the box, while communication apps such as Microsoft Teams, Outlook and WhatsApp are read.
  • Left-click or right-click the tray icon to open the menu.

Download / Build

This is a Windows GUI app and is built by GitHub Actions on a Windows runner (the authoritative build), because it uses Win32 / WinRT APIs that can’t be compiled on Linux/macOS.

  1. Push to main (or use Actions → Run workflow). The workflow compiles a release build and uploads PortableNotificationReader.exe (and a zip) as a build artifact — download it from the newest run’s Artifacts section.
  2. To cut a versioned release, push a tag like v1.0.0; the same binary is attached to a GitHub Release.

To build locally on a Windows machine with the Rust toolchain:

rustup target add x86_64-pc-windows-msvc
cargo build --release
# -> target\release\PortableNotificationReader.exe

⚠️ “Windows protected your PC” (SmartScreen)

When you first run the downloaded .exe, Windows may show a blue “Windows protected your PC” dialog from Microsoft Defender SmartScreen.

This is expected and the app is safe to run. SmartScreen flags any program that isn’t signed with a paid code-signing certificate (US$200–500 per year). The warning is about the missing signature, not anything the app does — the full source is in this repository.

How to run it anyway

  1. On the blue dialog, click More info.
  2. Click the Run anyway button that appears.

You only have to do this once per download.

Verify your download first (recommended)

Every release publishes the SHA-256 hash of the exe. Confirm your copy matches:

Get-FileHash .\PortableNotificationReader.exe -Algorithm SHA256

Compare the output to the hash on the Releases page.

Usage

  1. Put PortableNotificationReader.exe in any folder and run it. A tray icon appears; a small window near the tray shows the app’s icon so you know what to look for, then closes itself. config.json is created next to the exe on first save.
  2. Click the tray icon and pick a Voice, set Volume/Speed, and toggle Read notifications, Speak emojis, Pause during calls/meetings and Lower other apps while speaking on/off.
  3. Use Mute apps to mute specific applications, and the top-level Filter messages… / Filter and replace text… items to block/allow notifications or rewrite text before speaking.
Tip: Windows only records a toast in the database when notifications for that app are enabled in Settings → System → Notifications. If nothing is being read, first check the sending app is allowed to show notifications — then open Open diagnostics log to see exactly what the reader saw.

Autostart (optional)

Because the app is portable it doesn’t register itself. To start it with Windows, drop a shortcut to the .exe into your Startup folder (Win+R → shell:startup).

config.json

Created next to the executable. Example:

{
  "enabled": true,
  "selected_voice_id": "online:en-ZA-LeahNeural",
  "rate": 0,
  "volume": 130,
  "show_all_languages": false,
  "speak_emojis": false,
  "pause_on_mic": true,
  "duck_while_speaking": true,
  "muted_apps": ["Windows Security", "Snipping Tool", "immersivecontrolpanel"],
  "known_apps": ["WhatsApp", "Microsoft Teams", "Mail"],
  "filters": [
    { "pattern": "verification code", "is_regex": false, "block": true },
    { "pattern": "^Reminder:", "is_regex": true, "block": true }
  ],
  "replacements": [
    { "pattern": "AI", "replacement": "Artificial Intelligence", "is_regex": false },
    { "pattern": "\\[ERROR\\]", "replacement": "", "is_regex": true }
  ],
  "poll_interval_ms": 1000
}
FieldMeaning
selected_voice_idonline:{ShortName} for a neural voice, or sapi:{name} for an offline voice.
rateSpeaking speed, -10..10 (0 = normal).
volume0..200. 100 = normal; above 100 amplifies (louder than system).
show_all_languagesfalse = only voices matching the Windows display language.
speak_emojisfalse (default) strips emojis; true speaks their meanings.
pause_on_mictrue (default) keeps the app silent while on a call/in a meeting (any app using the microphone or camera). Set false to always read.
duck_while_speakingtrue (default) briefly lowers other apps’ volume while a notification is read. Set false to leave other audio untouched.
poll_interval_msHow often the notification DB is polled (minimum 250 ms; 1000 ms recommended).
filters[].blocktrue blocks matching notifications; false switches to allow-list mode.
replacements[]Text filtering & replacement rules. Empty replacement removes the match.

Filter rules match against the app name and the notification text. Replacements are applied after filters, in order, before speaking.
